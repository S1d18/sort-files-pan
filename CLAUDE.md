## Описание проекта

Утилита для автоматической сортировки файлов (PDF и изображения) с сетевого диска. Проект анализирует файлы по магическим сигнатурам, извлекает текст из PDF для классификации по ключевым словам (счета, билеты и т.д.) и раскладывает по соответствующим папкам.

## Команды

### Запуск полного пайплайна
```bash
python main.py
```

### Запуск отдельных этапов
```bash
python main.py --only type     # только исправление расширений
python main.py --only images   # только сортировка картинок
python main.py --only pdfs     # только классификация PDF
```

### Пропуск этапов
```bash
python main.py --skip-type --skip-images   # только PDF
python main.py --quiet                      # тихий режим
```

### Установка зависимостей
```bash
pip install -r requirements.txt
```

## Архитектура

### Главный пайплайн (main.py)
Последовательно запускает три этапа:
1. **run_type.py** — определяет реальный тип файлов без расширения по магическим сигнатурам, добавляет расширения
2. **run_images.py** — перемещает изображения в папку `Files* (img_file)`
3. **run_pdfs_multi.py** — многопроцессная классификация PDF по содержимому

### Модули
- **config.py** — все настройки: пути, метки папок, списки ключевых слов (`wordList`, `wordListMusor`), таймауты
- **scanner.py** — итераторы по файлам в корневых директориях с фильтрацией по расширениям
- **mover.py** — безопасное перемещение файлов с обработкой конфликтов имён (increment/hash/overwrite)
- **pdf_text.py** — извлечение текста из PDF с адаптивным выбором между быстрым путём и подпроцессом с таймаутом; поддерживает PyMuPDF (fitz) и pdfminer
- **text_rules.py** — извлечение дат из текста, проверка вхождения ключевых слов с Unicode-нормализацией

### Классификация PDF (run_pdfs_multi.py)
PDF классифицируются в папки:
- `Files* (block_pdf)` — защищённые паролем
- `Files* (file_error)` — не удалось прочитать
- `Files* (img_pdf)` — PDF без текста (сканы)
- `Files* (bill_musor_pdf)` — содержат "мусорные" слова (билеты, продукты)
- `Files* (bill_pdf)` — счета (по wordList), с подпапками по году
- `Files* (text_pdf(nokey))` — есть текст, но нет ключевых слов

### Вспомогательные скрипты
- **tg.py** — отправка уведомления в Telegram по завершении
- **clean_temp.py** — очистка Windows Temp от старых файлов

## Конфигурация (config.py)

Ключевые параметры:
- `ROOTS_GLOB` — glob-паттерн для поиска корневых папок (по умолчанию `M:\Files*`)
- `PDF_WORKERS` — количество процессов для обработки PDF (0 = авто)
- `PDF_PER_FILE_TIMEOUT_SEC` — таймаут на один PDF файл
- `MAX_PAGES_FOR_TEXT` — сколько страниц PDF анализировать
- `wordList` / `wordListMusor` — списки ключевых слов для классификации

## Особенности

- Многопроцессная обработка PDF с защитой от зависания (spawn-контекст, таймауты)
- Файлы, вызвавшие проблемы, записываются в `pdf_badlist.json` для последующего safe-mode
- SpeedTracker выводит прогресс каждые N файлов с ETA
- Поддержка Windows (лимит 61 воркер для ProcessPoolExecutor)
